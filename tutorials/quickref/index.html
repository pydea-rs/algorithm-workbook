Regex

~7 min · Python re
Import once: import re. Always use raw strings — r"\d+" — so backslashes reach the engine.

The re functions

re.search(p, s)First match anywhere → Match or None.
re.match(p, s)Match only at the start of s.
re.fullmatch(p, s)Whole string must match.
re.findall(p, s)List of strings (or tuples if >1 group).
re.finditer(p, s)Iterator of Match objects — use for positions.
re.sub(p, r, s)Replace all; r may be a function.
re.split(p, s)Split on the pattern.
re.compile(p)Pre-compile to reuse: rx.search(s).
Truthy test the result: if re.search(p, s): — never == True. findall returns groups, so wrap in (?:...) when you don't want capture.

Match object

m.group() / m[0]The whole match.
m.group(1) / m[1]First captured group.
m.groups()Tuple of all groups.
m.groupdict()Named groups → dict.
m.start() m.end()Index bounds of the match.
m.span()(start, end) tuple.

Character classes

.Any char except newline (\n).
\d \DDigit [0-9] / non-digit.
\w \WWord [A-Za-z0-9_] / non-word.
\s \SWhitespace / non-whitespace.
[abc]Any one of a, b, c.
[^abc]Any char except a, b, c.
[a-z0-9]Ranges combine inside a class.
\b \BWord boundary / non-boundary.

Quantifiers & anchors

*0 or more.
+1 or more.
?0 or 1 (optional).
{m} {m,n} {m,}Exactly m / between / at least.
*? +? ??Lazy — match as few as possible.
^ $Start / end (of line with re.M).
\A \ZStart / end of whole string.
a|bAlternation (a or b).
Greedy vs lazy: <.*> on "<a><b>" grabs the whole thing; <.*?> stops at the first >.

Groups & lookaround

(...)Capturing group.
(?:...)Group without capturing.
(?P<name>...)Named group → m["name"].
\1 (?P=name)Backreference to a group.
(?=...)Lookahead — followed by.
(?!...)Negative lookahead — not followed by.
(?<=...)Lookbehind — preceded by.
(?<!...)Negative lookbehind.

Flags

re.I / re.IGNORECASECase-insensitive.
re.M / MULTILINE^ $ match each line.
re.S / DOTALL. also matches newline.
re.X / VERBOSEIgnore whitespace, allow # comments.
re.A / ASCII\w \d \s = ASCII only.
flags=re.I|re.MCombine with |. Inline: (?im).

Copy-ready snippets

# all integers (with sign) in a string
nums = [int(x) for x in re.findall(r"-?\d+", s)]

# collapse runs of whitespace to one space, then trim
clean = re.sub(r"\s+", " ", s).strip()

# sub with a function — increment every number
out = re.sub(r"\d+", lambda m: str(int(m.group()) + 1), s)

# named groups → dict
m = re.match(r"(?P<y>\d{4})-(?P<mo>\d{2})-(?P<d>\d{2})", "2026-07-13")
m.groupdict()          # {'y':'2026','mo':'07','d':'13'}

# split on any non-word run (tokenize)
words = re.findall(r"[A-Za-z']+", text.lower())

# loose email / word@host.tld
re.findall(r"[\w.+-]+@[\w-]+\.[\w.-]+", s)

Handy tricks

(\w)\1Find a doubled char (backref).
re.escape(s)Quote user text before embedding in a pattern.
re.sub(p,r,s,count=1)Replace only the first N.
\g<name> \g<1>Reference groups in the replacement string.
(?i:abc)Scoped inline flag (3.6+).
re.subn(...)Like sub but returns (result, n).

VERBOSE for readable patterns

rx = re.compile(r"""
    (\d{4})   # year
    -(\d{2})  # month
    -(\d{2})  # day
""", re.X)
Gotchas: lookbehind must be fixed-width. Avoid nested quantifiers like (a+)+ on long input — catastrophic backtracking. Anchor with fullmatch when validating.

SQL

~8 min · standard + dialect notes
One of the three interview tasks is SQL/DB. This is the syntax card — the SQL Deep Dive module has the reasoning.

Query skeleton & logical order

SELECT   col, AGG(x)          # 5. pick / compute columns
FROM     t                    # 1. source rows
JOIN     u ON u.tid = t.id    # 1. combine
WHERE    t.active             # 2. filter ROWS (no aggregates)
GROUP BY col                  # 3. collapse into groups
HAVING   COUNT(*) > 1         # 4. filter GROUPS (aggregates ok)
ORDER BY AGG(x) DESC          # 6. sort
LIMIT 10 OFFSET 20;           # 7. page
Why it matters: WHERE runs before GROUP BY (can't reference aggregates); HAVING runs after (can). A column alias from SELECT isn't visible in WHERE.

Joins

INNER JOINOnly matching rows on both sides.
LEFT JOINAll left rows; NULLs where no right match.
RIGHT / FULLMirror / both sides kept.
CROSS JOINCartesian product.
self-joinJoin a table to itself with aliases.
Anti-join (rows with no match): LEFT JOIN … WHERE u.id IS NULL. Putting that filter in ON vs WHERE gives different results!

Aggregation

COUNT(*)All rows (incl. NULLs).
COUNT(col)Non-NULL values only.
COUNT(DISTINCT c)Unique non-NULL.
SUM AVG MIN MAXIgnore NULLs.
GROUP BY a, bOne row per distinct combo.

NULL logic

x IS NULLNever x = NULL (that's UNKNOWN).
COALESCE(a,b,0)First non-NULL.
NULLIF(a,b)NULL if a=b (avoid ÷0).
NOT IN (…NULL…)Returns 0 rows — use NOT EXISTS.

Window functions — "top N per group", running totals

# highest-paid employee per department
SELECT * FROM (
  SELECT e.*,
         ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS rn
  FROM employee e
) t
WHERE rn = 1;

# running total, and each row vs the previous
SELECT d,
       SUM(amount) OVER (ORDER BY d) AS running,
       LAG(amount) OVER (ORDER BY d) AS prev
FROM sales;
ROW_NUMBER()1,2,3,4 — always unique.
RANK()1,2,2,4 — ties share, gaps after.
DENSE_RANK()1,2,2,3 — ties share, no gap.
LAG(c) / LEAD(c)Previous / next row's value.
SUM() OVER (…)Aggregate without collapsing rows.

CTEs, subqueries, CASE

WITH big AS (
  SELECT dept_id, AVG(salary) a
  FROM employee GROUP BY dept_id
)
SELECT e.name
FROM employee e JOIN big ON big.dept_id = e.dept_id
WHERE e.salary > big.a;          # above dept avg

SELECT name,
  CASE WHEN salary > 100 THEN 'high'
       WHEN salary > 50  THEN 'mid'
       ELSE 'low' END AS band
FROM employee;

Set ops & strings

UNION / UNION ALLStack rows; ALL keeps duplicates (faster).
INTERSECT / EXCEPTCommon / difference.
a || bConcat (MySQL: CONCAT(a,b)).
LIKE '%x_'%=any run, _=one char.
DISTINCTDedup the whole row.

Recursive CTE (hierarchy / series)

WITH RECURSIVE nums(n) AS (
  SELECT 1
  UNION ALL
  SELECT n + 1 FROM nums WHERE n < 10
)
SELECT n FROM nums;
Same shape walks a manager → report tree: seed with the root, recurse on parent_id = n.id.

Dialect differences

LimitLIMIT n (PG/MySQL/SQLite); TOP n / FETCH FIRST (SQL Server/Oracle).
String concat|| standard; CONCAT() in MySQL (|| = OR there).
Case-insensitiveILIKE (PG); LOWER() both sides elsewhere.
UpsertON CONFLICT DO UPDATE (PG/SQLite); ON DUPLICATE KEY UPDATE (MySQL).
NowCURRENT_DATE, NOW(), EXTRACT(YEAR FROM d).
Agg filterCOUNT(*) FILTER (WHERE …) (PG) vs SUM(CASE WHEN … THEN 1 ELSE 0 END) (portable).

Algorithm Templates

~8 min · copy, then adapt
Skeletons you should be able to type without thinking. Read once so the shape is loaded; copy in the interview and fill the blanks.

Binary search (lower bound)

lo, hi = 0, len(a)          # [lo, hi)
while lo < hi:
    mid = (lo + hi) // 2
    if a[mid] < target:
        lo = mid + 1
    else:
        hi = mid
return lo   # first index with a[i] >= target
Same frame for "search on the answer": replace a[mid] < target with a feasible(mid) predicate.

Sliding window (longest valid)

from collections import Counter
cnt = Counter(); L = 0; best = 0
for R, c in enumerate(s):
    cnt[c] += 1
    while # window invalid: len(cnt) > k:
        cnt[s[L]] -= 1
        if cnt[s[L]] == 0: del cnt[s[L]]
        L += 1
    best = max(best, R - L + 1)

Two pointers

i, j = 0, len(a) - 1
while i < j:
    tot = a[i] + a[j]
    if tot == target: return (i, j)
    if tot < target: i += 1
    else: j -= 1

BFS on a grid

from collections import deque
R, C = len(grid), len(grid[0])
q = deque([(sr, sc)]); seen = {(sr, sc)}
while q:
    r, c = q.popleft()
    for dr, dc in ((1,0),(-1,0),(0,1),(0,-1)):
        nr, nc = r+dr, c+dc
        if 0<=nr<R and 0<=nc<C and (nr,nc) not in seen \
           and grid[nr][nc] != '#':
            seen.add((nr,nc)); q.append((nr,nc))

DFS & backtracking

res = []
def backtrack(start, path):
    res.append(path[:])              # record
    for i in range(start, len(nums)):
        path.append(nums[i])         # choose
        backtrack(i + 1, path)       # explore
        path.pop()                   # un-choose
backtrack(0, [])

Union-Find (DSU)

par = list(range(n))
def find(x):
    while par[x] != x:
        par[x] = par[par[x]]         # path compress
        x = par[x]
    return x
def union(a, b):
    par[find(a)] = find(b)

Dijkstra (weighted shortest path)

import heapq
dist = {s: 0}; pq = [(0, s)]
while pq:
    d, u = heapq.heappop(pq)
    if d > dist.get(u, INF): continue
    for v, w in graph[u]:
        nd = d + w
        if nd < dist.get(v, INF):
            dist[v] = nd
            heapq.heappush(pq, (nd, v))

Topological sort (Kahn)

from collections import deque
indeg = [0]*n
for u in range(n):
    for v in adj[u]: indeg[v] += 1
q = deque(u for u in range(n) if indeg[u]==0)
order = []
while q:
    u = q.popleft(); order.append(u)
    for v in adj[u]:
        indeg[v] -= 1
        if indeg[v] == 0: q.append(v)
# len(order) < n  ->  cycle exists

DP — 1D (coin change)

dp = [0] + [INF]*amount
for a in range(1, amount+1):
    for c in coins:
        if c <= a:
            dp[a] = min(dp[a], dp[a-c] + 1)
return dp[amount] if dp[amount] < INF else -1

Monotonic stack (next greater)

res = [-1]*len(a); st = []          # holds indices
for i, x in enumerate(a):
    while st and a[st[-1]] < x:
        res[st.pop()] = x
    st.append(i)

Trie (prefix tree)

root = {}
def insert(w):
    node = root
    for c in w:
        node = node.setdefault(c, {})
    node['$'] = True                 # end mark
def search(w):
    node = root
    for c in w:
        if c not in node: return False
        node = node[c]
    return '$' in node

Quickselect (k-th smallest, avg O(n))

import random
def qselect(a, k):                   # k is 0-based
    p = random.choice(a)
    lo = [x for x in a if x < p]
    hi = [x for x in a if x > p]
    eq = len(a) - len(lo) - len(hi)
    if k < len(lo):      return qselect(lo, k)
    if k < len(lo)+eq:   return p
    return qselect(hi, k - len(lo) - eq)
Or just heapq.nsmallest(k, a)[-1] when clarity beats optimality.

Complexity & Gotchas

~4 min · state it out loud
Interviewers grade whether you name the complexity. And these Python traps cost the most silent debugging time.

Cost of common operations

list[i], .appendO(1) — append amortized.
list.insert(0), .pop(0)O(n) — use deque.
x in listO(n) — use a set.
x in set/dict, get/addO(1) average.
deque both endsO(1).
heappush / heappopO(log n).
sorted()O(n log n).
bisectO(log n) search, O(n) insert.

Time budget (~10⁸ ops/sec)

n ≤ 10O(n!) / O(2ⁿ) backtracking fine.
n ≤ 20O(2ⁿ) bitmask DP.
n ≤ 500O(n³) ok.
n ≤ 5,000O(n²) ok.
n ≤ 10⁶O(n log n) / O(n).
n > 10⁷O(n) or better, watch constants.
Reading the constraints first tells you the target complexity before you design.

Python traps that eat interview time

# 1. Mutable default arg — shared across calls!
def f(acc=[]):     # BUG: same list every call
def f(acc=None):   # FIX
    if acc is None: acc = []

# 2. Integer division floors toward -inf
-7 // 2   ==  -4          7 / 2  == 3.5   # / is float

# 3. Copy vs reference
b = a[:]           # shallow copy of a list
grid = [[0]*C for _ in range(R)]   # NOT [[0]*C]*R

# 4. String build in a loop is O(n^2)
"".join(parts)     # not  s += piece

# 5. Don't mutate a list while iterating it
xs = [x for x in xs if keep(x)]    # rebuild instead
Also: is tests identity, == tests value — compare with == (except is None). Default recursion limit ≈ 1000; raise it for deep DFS or go iterative.
