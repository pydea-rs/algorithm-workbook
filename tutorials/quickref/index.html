Regex

#~7 min · Python re
Import once: import re. Always use raw strings — r"\d+" — so a backslash reaches the engine instead of being eaten by Python as an escape.

The re functions

re.search(p, s)First match found anywhere in s → Match or None.
re.match(p, s)Tries to match starting at index 0 only (not "whole string").
re.fullmatch(p, s)The pattern must consume the entire string — best for validation.
re.findall(p, s)All non-overlapping matches as a list of strings — but of the group if the pattern has one, or tuples if it has several.
re.finditer(p, s)Same matches as an iterator of Match objects — use when you need positions.
re.sub(p, r, s)Replace every match with r (a string or a function of the match).
re.split(p, s)Split s wherever the pattern matches.
re.compile(p)Compile once, reuse: rx = re.compile(p); rx.search(s).
Test the result for truthiness — if re.search(p, s): — never == True. Because findall returns groups, wrap parts you don't want captured in (?:...).

Match object

m.group() / m[0]The whole matched substring.
m.group(1) / m[1]Text captured by the 1st (...).
m.groups()Tuple of all captured groups.
m["name"]A named group; m.groupdict() → dict of all.
m.start() m.end()Index where the match begins / ends.
m.span()(start, end) as a tuple.

Character classes

.Any single char except newline (add re.S to include it).
\d \DA digit [0-9] / anything that is not a digit.
\w \WA word char [A-Za-z0-9_] / a non-word char.
\s \SAny whitespace (space, tab, newline) / non-whitespace.
[abc]Exactly one of the listed characters.
[^abc]One character that is none of these.
[a-z0-9]Ranges; combine several in one class.
\b \BZero-width boundary between a \w and a non-\w / its opposite.

Quantifiers & anchors

*The previous item, 0 or more times.
+1 or more times.
?0 or 1 time (makes it optional).
{m} {m,n} {m,}Exactly m / between m and n / at least m.
*? +? ??Lazy — match as few chars as possible (default is greedy).
^ $Start / end of the string — of each line under re.M.
\A \ZStart / end of the whole string, always (ignores re.M).
a|bMatch a or b (alternation).
Greedy vs lazy: on "<a><b>", <.*> grabs the whole string (greedy), while <.*?> stops at the first > and matches just <a>.

Groups & lookaround

(...)Group and capture the matched text.
(?:...)Group without capturing (keeps findall/numbers clean).
(?P<name>...)Capture into a name → read via m["name"].
\1 (?P=name)Backreference — match the same text a group already captured.
(?=...)Lookahead: assert what follows, without consuming it.
(?!...)Negative lookahead: assert what does not follow.
(?<=...)Lookbehind: assert what precedes (must be fixed width).
(?<!...)Negative lookbehind.
Lookarounds are zero-width: they check a condition but don't move the cursor, so several can stack at one position (see the password example below).

Flags

re.I / IGNORECASEMatch regardless of upper/lower case.
re.M / MULTILINEMake ^ and $ match at every line break.
re.S / DOTALLLet . also match newline characters.
re.X / VERBOSEIgnore whitespace in the pattern and allow # comments.
re.A / ASCIIRestrict \w \d \s to ASCII (off = Unicode).
combineflags=re.I|re.M, or inline at the front: (?im).

Copy-ready snippets

# every integer (optional sign) pulled out of a string
nums = [int(x) for x in re.findall(r"-?\d+", s)]

# collapse any run of whitespace to one space, then trim the ends
clean = re.sub(r"\s+", " ", s).strip()

# replace using a function — bump every number by one
out = re.sub(r"\d+", lambda m: str(int(m.group()) + 1), s)

# named groups -> dict
m = re.match(r"(?P<y>\d{4})-(?P<mo>\d{2})-(?P<d>\d{2})", "2026-07-13")
m.groupdict()          # {'y':'2026','mo':'07','d':'13'}

# tokenize words (letters and apostrophes only)
words = re.findall(r"[A-Za-z']+", text.lower())

🧪 Worked examples

1 Validate an identifier: a letter or _, then letters/digits/_.
def is_ident(s):
    return re.fullmatch(r"[A-Za-z_]\w*", s) is not None
Why: fullmatch anchors both ends for you — no ^…$ needed.
2 Password rule: ≥ 8 chars with at least one lower, upper and digit.
rx = re.compile(r"^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$")
bool(rx.match("Abcd1234"))      # True
Why: each (?=…) re-scans from the start; order is irrelevant, and nothing is consumed until .{8,}.
3 Reformat "John Doe" → "Doe, John".
re.sub(r"(\w+)\s+(\w+)", r"\2, \1", "John Doe")   # 'Doe, John'
Why: \1 \2 in the replacement re-emit the captured groups, swapped.
4 Find an accidentally doubled word ("the the").
re.findall(r"\b(\w+)\s+\1\b", "the the cat", re.I)   # ['the']
Why: the backreference \1 forces the second word to equal the first.

Handy tricks

re.escape(s)Quote user text before dropping it into a pattern.
re.sub(p,r,s,count=1)Stop after N replacements.
\g<name> \g<1>Reference groups inside the replacement string.
(?i:abc)Turn a flag on for just part of the pattern (3.6+).
re.subn(...)Like sub but returns (result, how_many).

VERBOSE for readable patterns

rx = re.compile(r"""
    (\d{4})   # year
    -(\d{2})  # month
    -(\d{2})  # day
""", re.X)
Gotchas: lookbehind must be fixed-width ((?<=ab) ok, (?<=a+) not). Avoid nesting quantifiers like (a+)+ on long text — it triggers catastrophic backtracking. Validate with fullmatch, not search.

SQL

#~8 min · standard + dialect notes
One of the three interview tasks is SQL/DB. This is the syntax card — the SQL Deep Dive module has the reasoning behind it.

Query skeleton & logical order

SELECT   col, AGG(x)          # 5. pick / compute the output columns
FROM     t                    # 1. the source rows
JOIN     u ON u.tid = t.id    # 1. glue tables together
WHERE    t.active             # 2. filter individual ROWS (no aggregates yet)
GROUP BY col                  # 3. collapse rows into one per group
HAVING   COUNT(*) > 1         # 4. filter GROUPS (aggregates allowed here)
ORDER BY AGG(x) DESC          # 6. sort the result
LIMIT 10 OFFSET 20;           # 7. paginate
Why the order matters: WHERE runs before grouping, so it can't see aggregates; HAVING runs after, so it can. And a SELECT alias isn't visible in WHERE — the SELECT is computed almost last.

Joins

INNER JOINOnly rows that match on both sides.
LEFT JOINEvery left row; right columns are NULL when there's no match.
RIGHT / FULLMirror of LEFT / keep unmatched rows from both sides.
CROSS JOINEvery combination (Cartesian product).
self-joinJoin a table to itself with two aliases (e.g. employee ↔ manager).
Anti-join (rows with no match): LEFT JOIN … WHERE u.id IS NULL. Note a condition in ON filters before the join, in WHERE after — different results.

Aggregation

COUNT(*)Number of rows in the group, NULLs included.
COUNT(col)Rows where col is not NULL.
COUNT(DISTINCT c)Distinct non-NULL values.
SUM AVG MIN MAXSkip NULLs; AVG divides by the non-NULL count.
GROUP BY a, bOne output row per distinct (a, b).

NULL logic

x IS NULLThe only correct test — x = NULL is always UNKNOWN.
COALESCE(a,b,0)First argument that isn't NULL.
NULLIF(a,b)NULL when a = b (handy to dodge divide-by-zero).
NOT IN (…NULL…)Yields zero rows if the list has a NULL — use NOT EXISTS.

Window functions — "top N per group", running totals

A window function computes across a set of rows without collapsing them — every input row still comes back, now carrying a rank or running value. PARTITION BY restarts the calculation per group; ORDER BY sets the order it walks.

# highest-paid employee in each department
SELECT * FROM (
  SELECT e.*,
         ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS rn
  FROM employee e
) t
WHERE rn = 1;

# running total, and each row compared with the previous one
SELECT d,
       SUM(amount) OVER (ORDER BY d) AS running,
       LAG(amount) OVER (ORDER BY d) AS prev
FROM sales;
ROW_NUMBER()1,2,3,4 — always distinct, even on ties.
RANK()1,2,2,4 — ties share a rank, then it skips.
DENSE_RANK()1,2,2,3 — ties share, no gap after.
LAG(c) / LEAD(c)Value from the previous / next row.
SUM() OVER (…)Aggregate as a running/partition total, rows kept.

CTEs, subqueries, CASE

WITH big AS (
  SELECT dept_id, AVG(salary) AS a
  FROM employee GROUP BY dept_id
)
SELECT e.name
FROM employee e JOIN big ON big.dept_id = e.dept_id
WHERE e.salary > big.a;          # paid above their dept average

SELECT name,
  CASE WHEN salary > 100 THEN 'high'
       WHEN salary > 50  THEN 'mid'
       ELSE 'low' END AS band
FROM employee;

A WITH block (CTE) just names a subquery so the main query reads top-to-bottom.

Set ops & strings

UNION / UNION ALLStack two result sets; ALL keeps duplicates and is faster.
INTERSECT / EXCEPTRows in both / in the first but not the second.
a || bConcatenate (MySQL uses CONCAT(a,b)).
LIKE '%x_'% = any run of chars, _ = exactly one.
DISTINCTDrop duplicate rows from the result.

🧪 Worked examples

1 Second-highest salary (generalises to the N-th).
SELECT DISTINCT salary FROM (
  SELECT salary,
         DENSE_RANK() OVER (ORDER BY salary DESC) AS r
  FROM employee
) t
WHERE r = 2;
Why: DENSE_RANK so equal salaries don't burn a rank; change 2 to N.
2 Employees who earn more than their manager.
SELECT e.name
FROM employee e
JOIN employee m ON e.manager_id = m.id
WHERE e.salary > m.salary;
Why: the same table, aliased twice — e is the worker, m the manager.
3 Month-over-month change, no self-join.
SELECT month,
       revenue - LAG(revenue) OVER (ORDER BY month) AS delta
FROM monthly;
Why: LAG reaches back one row; the first row's delta is NULL.
4 Delete duplicate rows, keep the smallest id per email.
DELETE FROM person
WHERE id NOT IN (
  SELECT MIN(id) FROM person GROUP BY email
);
Why: group by the "sameness" key, keep one representative, drop the rest.

Recursive CTE (series / hierarchy)

WITH RECURSIVE nums(n) AS (
  SELECT 1                       # anchor: the seed row
  UNION ALL
  SELECT n + 1 FROM nums WHERE n < 10   # recurse
)
SELECT n FROM nums;
The same shape walks a manager → report tree: seed with the root, then recurse on parent_id = n.id.

Dialect differences

LimitLIMIT n (PG/MySQL/SQLite); TOP n or FETCH FIRST n ROWS (SQL Server/Oracle).
Concat|| is standard; in MySQL || means OR — use CONCAT() there.
Case-insensitiveILIKE (Postgres); elsewhere LOWER(a) = LOWER(b).
UpsertON CONFLICT DO UPDATE (PG/SQLite); ON DUPLICATE KEY UPDATE (MySQL).
Conditional countCOUNT(*) FILTER (WHERE …) (PG) vs the portable SUM(CASE WHEN … THEN 1 ELSE 0 END).

🧪 Worked examples

1 Group words that are anagrams of each other.
from collections import defaultdict
g = defaultdict(list)
for w in words:
    g[tuple(sorted(w))].append(w)   # sorted letters = shared key
groups = list(g.values())
Why: anagrams collapse to the same sorted-letter key — no pairwise comparison.
2 The k most frequent elements.
from collections import Counter
Counter(nums).most_common(k)        # [(val, freq), …]
# tie-agnostic / big data: heapq.nlargest(k, cnt, key=cnt.get)
Why: Counter does the tally; most_common does the partial sort.
3 Merge overlapping intervals.
intervals.sort()                    # by start
out = [intervals[0]]
for s, e in intervals[1:]:
    if s <= out[-1][1]:             # overlaps the last kept
        out[-1][1] = max(out[-1][1], e)
    else:
        out.append([s, e])
Why: after sorting by start, you only ever compare with the last merged interval.

Algorithm Templates

#~9 min · copy, then adapt
Skeletons you should be able to type without thinking. Read once so the shape is loaded; the worked examples at the end show two of them plugged into real problems.

Binary search (lower bound)

lo, hi = 0, len(a)          # search the half-open range [lo, hi)
while lo < hi:
    mid = (lo + hi) // 2
    if a[mid] < target:
        lo = mid + 1
    else:
        hi = mid
return lo   # first index with a[i] >= target
Same frame solves "search on the answer": swap a[mid] < target for a feasible(mid) predicate and binary-search the smallest feasible value.

Sliding window (longest valid)

from collections import Counter
cnt = Counter(); L = 0; best = 0
for R, c in enumerate(s):
    cnt[c] += 1
    while len(cnt) > k:              # shrink while the window is invalid
        cnt[s[L]] -= 1
        if cnt[s[L]] == 0:
            del cnt[s[L]]
        L += 1
    best = max(best, R - L + 1)
This one finds the longest substring with at most k distinct chars. The invariant lives in the while; everything else is boilerplate.

Two pointers

i, j = 0, len(a) - 1        # a is sorted
while i < j:
    tot = a[i] + a[j]
    if tot == target:
        return (i, j)
    if tot < target:
        i += 1              # need bigger -> move left up
    else:
        j -= 1

BFS on a grid

from collections import deque
R, C = len(grid), len(grid[0])
q = deque([(sr, sc)]); seen = {(sr, sc)}
while q:
    r, c = q.popleft()
    for dr, dc in ((1,0),(-1,0),(0,1),(0,-1)):
        nr, nc = r + dr, c + dc
        if 0 <= nr < R and 0 <= nc < C \
           and (nr, nc) not in seen and grid[nr][nc] != '#':
            seen.add((nr, nc)); q.append((nr, nc))

DFS & backtracking

res = []
def backtrack(start, path):
    res.append(path[:])              # record a copy of this state
    for i in range(start, len(nums)):
        path.append(nums[i])         # choose
        backtrack(i + 1, path)       # explore
        path.pop()                   # un-choose (backtrack)
backtrack(0, [])

Union-Find (DSU)

par = list(range(n))
def find(x):
    while par[x] != x:
        par[x] = par[par[x]]         # path compression
        x = par[x]
    return x
def union(a, b):
    par[find(a)] = find(b)           # merge the two sets

Topological sort (Kahn)

from collections import deque
indeg = [0]*n
for u in range(n):
    for v in adj[u]:
        indeg[v] += 1
q = deque(u for u in range(n) if indeg[u] == 0)
order = []
while q:
    u = q.popleft(); order.append(u)
    for v in adj[u]:
        indeg[v] -= 1
        if indeg[v] == 0:
            q.append(v)
# len(order) < n  ->  the graph has a cycle

DP — 1D (coin change)

INF = float('inf')
dp = [0] + [INF]*amount              # dp[a] = fewest coins for a
for a in range(1, amount + 1):
    for c in coins:
        if c <= a:
            dp[a] = min(dp[a], dp[a - c] + 1)
return dp[amount] if dp[amount] < INF else -1

Monotonic stack (next greater)

res = [-1]*len(a); st = []           # st holds indices, values decreasing
for i, x in enumerate(a):
    while st and a[st[-1]] < x:
        res[st.pop()] = x            # x is the next-greater for that index
    st.append(i)

🧪 Worked examples — templates plugged into real problems

1 Number of islands (flood-fill with the grid template).
def num_islands(grid):
    R, C = len(grid), len(grid[0]); seen = set(); n = 0
    for i in range(R):
        for j in range(C):
            if grid[i][j] == '1' and (i, j) not in seen:
                n += 1; st = [(i, j)]; seen.add((i, j))
                while st:                       # flood this island
                    r, c = st.pop()
                    for dr, dc in ((1,0),(-1,0),(0,1),(0,-1)):
                        nr, nc = r + dr, c + dc
                        if 0 <= nr < R and 0 <= nc < C \
                           and grid[nr][nc] == '1' and (nr, nc) not in seen:
                            seen.add((nr, nc)); st.append((nr, nc))
    return n
Why: each unvisited '1' starts one flood fill; the number of fills is the answer.
2 Detect a cycle in a directed graph (Kahn's template).
from collections import deque
def has_cycle(n, adj):
    indeg = [0]*n
    for u in range(n):
        for v in adj[u]:
            indeg[v] += 1
    q = deque(u for u in range(n) if indeg[u] == 0)
    seen = 0
    while q:
        u = q.popleft(); seen += 1
        for v in adj[u]:
            indeg[v] -= 1
            if indeg[v] == 0:
                q.append(v)
    return seen != n                # couldn't order every node -> cycle
Why: a topological order exists iff the graph is acyclic — this is "course schedule" in disguise.

Trie (prefix tree)

root = {}
def insert(w):
    node = root
    for c in w:
        node = node.setdefault(c, {})
    node['$'] = True                 # mark a complete word
def search(w):
    node = root
    for c in w:
        if c not in node:
            return False
        node = node[c]
    return '$' in node

Quickselect (k-th smallest, avg O(n))

import random
def qselect(a, k):                   # k is 0-based
    p = random.choice(a)
    lo = [x for x in a if x < p]
    hi = [x for x in a if x > p]
    eq = len(a) - len(lo) - len(hi)
    if k < len(lo):
        return qselect(lo, k)
    if k < len(lo) + eq:
        return p
    return qselect(hi, k - len(lo) - eq)
When clarity beats the last bit of speed, just heapq.nsmallest(k, a)[-1].

Bit Tricks

#~3 min · the ones that actually show up
Rare in day-to-day code, but they turn up in masks, subset enumeration and "do it without extra memory" questions. Python ints are arbitrary-precision, so no overflow to worry about.

Core operators & idioms

x & 1Test the lowest bit → odd (1) or even (0).
x << k · x >> kMultiply / floor-divide by 2**k.
x & (x - 1)Clears the lowest set bit — == 0 means power of two.
x & -xIsolates the lowest set bit (used in Fenwick trees).
x ^ yXOR; a ^ a == 0 and a ^ 0 == a.
(x >> k) & 1Read bit k.
x | (1 << k)Set bit k (clear = x & ~(1 << k), toggle = ^).
(1 << n) - 1A mask of n ones.
x.bit_count()Number of set bits (3.10+); else bin(x).count("1").

Reading & building masks

# treat an int as a set of items 0..n-1
mask = 0
mask |= (1 << i)          # add item i
mask &= ~(1 << i)         # remove item i
if mask & (1 << i): ...   # is item i in the set?

# enumerate the members of a mask
bits = [i for i in range(n) if mask >> i & 1]

full = (1 << n) - 1        # the "all items" set

🧪 Worked examples

1 The one number that appears once (all others appear twice).
from functools import reduce
reduce(lambda a, b: a ^ b, nums)    # XOR cancels the pairs
Why: equal values XOR to 0, so only the lonely one survives — O(1) memory.
2 Is x a power of two?
def is_pow2(x):
    return x > 0 and x & (x - 1) == 0
Why: a power of two has exactly one set bit; x & (x-1) removes it, leaving 0.
3 Iterate every submask of a mask (subset DP).
sub = mask
while sub:
    use(sub)
    sub = (sub - 1) & mask          # next smaller submask
# the empty submask (0) is handled after the loop
Why: (sub - 1) & mask walks submasks in descending order without repeats.

Complexity & Gotchas

#~5 min · state it out loud
Interviewers grade whether you name the complexity — and read the constraints to pick the target before you design. These Python traps cost the most silent debugging time.

Cost of common operations

list[i], .appendO(1) — append is amortized O(1).
list.insert(0), .pop(0)O(n) — shifts everything; use deque.
x in listO(n) scan — use a set for membership.
x in set/dict, get/addO(1) average.
deque both endsO(1) append/pop left and right.
heappush / heappopO(log n).
sorted()O(n log n).
bisectO(log n) to find, O(n) to insert (shifts).

Time budget (~10⁸ ops/sec)

Read the input size first — it tells you the intended complexity before you write a line.

n ≤ 10O(n!) / O(2ⁿ) — full brute force is fine.
n ≤ 20O(2ⁿ) bitmask DP over subsets.
n ≤ 500O(n³) is okay.
n ≤ 5,000O(n²) is okay.
n ≤ 10⁶O(n log n) or O(n).
n > 10⁷O(n) or better, and mind constant factors.

Python traps that eat interview time

# 1. Mutable default argument — shared across every call!
def f(acc=[]):        # BUG: the same list persists between calls
    ...
def f(acc=None):      # FIX
    if acc is None:
        acc = []

# 2. Integer division floors toward negative infinity
-7 // 2   ==  -4          7 / 2  ==  3.5    # / always gives a float

# 3. Copying vs aliasing
b = a[:]              # shallow copy of a list
grid = [[0]*C for _ in range(R)]   # NOT [[0]*C]*R (shared rows!)

# 4. Building a string in a loop is O(n^2)
"".join(parts)        # do this, not  s += piece

# 5. Never mutate a list while iterating it
xs = [x for x in xs if keep(x)]    # rebuild instead
Also: is compares identity, == compares value — use == for data, is only for None. Default recursion limit is ~1000; raise it or go iterative for deep DFS.

🧪 Worked examples — let the constraints choose the approach

1 n ≤ 1e5: does any pair sum to target?
# O(n^2) = 1e10 -> too slow. A hash set makes it one pass:
seen = set()
for x in nums:
    if target - x in seen:
        return True
    seen.add(x)
return False
Why: 10⁵ squared is ten billion ops; trading memory for time gives O(n).
2 n ≤ 20: the small bound is the hint — try every subset.
# 2^20 is about 1e6 subsets — exponential is intended here
for mask in range(1 << n):
    subset = [a[i] for i in range(n) if mask >> i & 1]
    consider(subset)
Why: when n is tiny, an "impossible" O(2ⁿ) is often exactly what's expected.
